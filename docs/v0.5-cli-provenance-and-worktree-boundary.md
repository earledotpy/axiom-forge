# v0.5 — CLI Provenance and Disposable-Worktree Boundary Evidence

## Completed

### 2026-06-21 — Record adapter CLI provenance

- `record.json` now uses schema version 2 and records `cli_command`,
  `cli_path`, and `cli_version`.
- Real CLI adapters capture the invoked command, resolved executable path, and
  first non-empty line of a best-effort `--version` invocation before running
  the task CLI.
- A version query that cannot produce a value records `cli_version: null`; it
  does not make an otherwise valid adapter run fail.
- A missing required CLI still fails closed as `agent_execution_failed` and
  records no fabricated provenance. Malformed provenance also fails closed as
  `cli_provenance_invalid`.
- The simulated adapter records the Python executable and version it uses, so
  the same record contract is exercised by the deterministic runner path.

### 2026-06-21 — Require standard adapter CLIs for the full health proof

- `scripts/forge_check.sh` runs `scripts/check_adapters.sh` before the gate,
  promotion, and runner matrices.
- `codex` and `claude` are required because they support the standard
  CLI-backed adapters; either missing command fails the preflight before the
  matrices run.
- `agy` is reported as optional because `antigravity` remains experimental.
  Its absence does not fail the health proof.
- This is an explicit local-environment precondition, not evidence of adapter
  trust or a substitute for captured run provenance.

### 2026-06-21 — Detect target-checkout writes outside the disposable worktree

- The runner snapshots target-checkout Git status, including ignored paths,
  immediately before and after adapter execution.
- A changed snapshot fails closed with
  `adapter_modified_outside_worktree` in the failed `record.json`.
- The check runs before adapter-exit handling, so an outside-worktree mutation
  receives that specific reason even when the adapter also exits nonzero.
- `bad-outside-worktree-agent` writes an ignored file in the parent checkout;
  runner cases R2d and R2e prove detection and cleanup.

## Scope Boundary

This records the v0.5 implementation and its local verification only. It does
not change adapter trust, adapter status, promotion policy, or treat captured
CLI metadata as proof of adapter safety. Target-checkout mutation detection is
not OS-level containment and does not observe arbitrary host-path writes.
