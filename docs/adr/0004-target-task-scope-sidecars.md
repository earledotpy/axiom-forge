# Enforce target task scope from Forge-owned sidecars

Target-mode tasks declare their target task scope in a Forge-owned sidecar file next to the task file, using the same basename and replacing `.task.md` with `.allowed-paths.txt`. The sidecar contains exact target-repository-relative file paths, one per line, with optional blank lines and `#` comments; absolute paths, traversal, globs, directory entries, and backslash-separated paths are invalid. Target-mode capture fails closed before adapter execution when the sidecar is missing, empty, or invalid, and copies the accepted sidecar into the captured run directory with its hash recorded in the run record. Target-mode verification enforces scope from the captured sidecar by checking the changed path set after applying the patch in a disposable verifier worktree, with renames requiring both old and new paths to be allowed and deletions allowed when the deleted path is listed. This keeps operator-controlled scope separate from agent-visible task text, makes scope part of Forge-owned evidence, and prevents a later task-sidecar edit from changing what an existing captured run is allowed to promote.
