# Committed Delegation and Review Evidence

Delegation artifacts must be committed to the Forge repository before an adapter run begins, and promotion review results must be committed before promotion. This trades some local iteration convenience for auditability: each captured run can point back to an exact approved revision of the runnable task file, approved path-scope sidecar, and operator-approved acceptance check that the adapter received, while each promotion can point back to a stable review result. The captured run record should identify the Forge commit SHA for its delegation artifacts, and the promotion record should identify the Forge commit SHA for the review result it relied on, so both delegation authority and promotion authority are reconstructable later.

`scripts/delegation_artifact_set.py` is the internal ownership seam for Delegation artifact set rules. It owns the checks that make a task a Delegation-ready task: the runnable task file must exist, the Target task scope sidecar must be valid and non-empty, the Operator-approved acceptance check must be present and non-empty, and the acceptance check must stay outside the approved target path scope. It also owns the target-mode copied-evidence rule by placing the approved scope in the captured run directory and recording its hash, and it owns committed artifact lookup by resolving acceptance content from the recorded Delegation artifact revision rather than from latest task-sidecar state.

The public command contracts do not move to this module. Operators and tests still use `scripts/run_agent_task.sh`, `scripts/verify_patch.sh`, and `scripts/promote.sh`; the module is an internal ownership point behind those shells. Target-mode verification must continue to use copied Forge-owned evidence from `runs/<run-id>/` and committed artifact lookup from the recorded revision. Historical run evidence is not migrated by this refactor, because old captured runs remain evidence of what Forge recorded at the time.
