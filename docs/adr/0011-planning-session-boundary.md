# Bound workbench planning sessions to learn-anything-change-nothing

The workbench will host interactive planning chat sessions with CLI agents (claude-code and codex first), which conflicts with reading ADR 0006 as "the workbench never talks to agents conversationally." We resolved this by adding a separately-bounded activity rather than superseding the execution cage: a planning session may read the target repository and Forge evidence and run non-mutating investigation commands in a disposable worktree, but can change nothing durable — its only outputs are its Forge-owned transcript and a draft proposal that pre-fills the operator approval form with zero authority. We rejected read-only sessions (they cripple investigation, the main reason to chat with an agent about code) and unrestrained sessions in quarantined workspaces (they rely on per-vendor sandboxing, exactly the implicit trust Axiom Forge exists to avoid). Qualification does not gate planning: an unqualified adapter may host a session, because trust is enforced at execution and promotion, not planning.
