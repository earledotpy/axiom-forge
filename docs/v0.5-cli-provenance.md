# v0.5 — CLI Provenance in Captured Run Records

## Completed

### 2026-06-21 — Record adapter CLI provenance

- `record.json` now uses schema version 2 and records `cli_command`,
  `cli_path`, and `cli_version`.
- Real CLI adapters capture the invoked command, resolved executable path, and
  first non-empty line of a best-effort `--version` invocation before running
  the task CLI.
- A version query that cannot produce a value records `cli_version: null`; it
  does not make an otherwise valid adapter run fail.
- A missing required CLI still fails closed as `agent_execution_failed` and
  records no fabricated provenance. Malformed provenance also fails closed as
  `cli_provenance_invalid`.
- The simulated adapter records the Python executable and version it uses, so
  the same record contract is exercised by the deterministic runner path.

## Verification

- `tests/runner/run_all.sh` passed in an isolated clean clone: 13 passed, 0
  failed.
- The runner coverage includes a completed record with populated provenance
  and a missing-CLI record with null provenance fields.
- `git diff --check` passed.

## Scope Boundary

This records the v0.5 implementation and its local verification only. It does
not change adapter trust, adapter status, promotion policy, or treat captured
CLI metadata as proof of adapter safety.
